# 腎安守護技術說明

## 作品定位

腎安守護是一套針對洗腎患者「離開透析室後」的返家與居家恢復期照護系統。設計重點不是取代醫療診斷，而是把患者、家屬、看護與居服員辦公室之間的照護資訊接起來，讓異常狀態能更早被發現。

## 三端式系統

### 1. 洗腎者佩戴裝置

功能：

- 顯示時間
- 顯示心率
- 顯示血壓
- 顯示跌倒偵測狀態

設計理由：

透析後常見疲倦、暈眩或血壓波動，患者返家途中可能還沒有家屬或照護者在旁。佩戴裝置讓患者端保有基本生命徵象與求救狀態顯示，降低離開透析室後的資訊斷點。

### 2. 床邊檢測器

功能：

- 追蹤活動狀態
- 異常時警報求救
- 支援床邊通話或看護遠端對話

設計理由：

患者回到家後，危險可能發生在休息、起身、走動或夜間活動時。床邊檢測器補足穿戴裝置之外的居家場域資訊，當活動量過低或疑似跌倒時，可以開啟通話並通知照護端。

### 3. 居服員辦公室看板

功能：

- 顯示多位被照顧者健康指數
- 標示低於門檻的個案
- 提示派員協助

設計理由：

居服員通常同時照顧多位個案，若只靠電話或家屬主動回報，容易延誤。辦公室看板把不同患者的狀態轉成同一套健康指數，使照護端能快速判斷誰需要優先協助。

## 資料輸入

原型示範使用四項輸入：

- 收縮壓 systolicBp
- 心率 heartRate
- 活動指數 activityIndex
- 跌倒偵測 fallDetected

對應實體感測器可包含：

- 血壓量測模組
- 心率或血氧感測器
- IMU 跌倒偵測模組
- 床邊毫米波、紅外線或活動感測器
- Wi-Fi / LTE / BLE 傳輸模組

## 健康指數邏輯

示範程式位於 `src/risk.js`。計算方式如下：

1. 將收縮壓、心率、活動指數與跌倒事件分別轉換為風險分數。
2. 將風險分數相加，得到總風險。
3. 健康指數 = 100 - 總風險。
4. 若偵測到跌倒，健康指數最高只保留到 20，並直接進入立即協助狀態。

分級：

- 66 至 100：穩定
- 41 至 65：觀察
- 21 至 40：派員確認
- 0 至 20 或跌倒偵測：立即協助

## 示範程式

網頁示範：

- `src/main.js`：Three.js 3D 模型、互動控制與畫面同步。
- `src/risk.js`：健康指數與照護分級邏輯。
- `src/styles.css`：響應式介面與投稿用版面。

硬體草稿：

- `firmware/renal_guardian_monitor.ino`：Arduino 風格感測與派員邏輯草稿。

可執行指令：

```bash
npm install
npm run dev -- --port 5173
npm run build
npm run export:glb
```

## 投稿輸出

主要投稿包：

驗證：

```bash
npm run validate:submission
```

## 原型限制

- 目前為概念原型，數值由互動滑桿模擬。
- 健康指數為示範用規則模型，正式醫療產品需由臨床資料校正。
- 實際部署需要資料加密、權限控管、設備校準與醫療/照護流程驗證。
